# Razorpay Test Mode Integration - Checkout Page

## Goal
Integrate Razorpay payment gateway in **test mode** on `checkout.php` so UPI and CARD payments go through Razorpay's checkout modal, while COD remains unchanged.

## Current State
- No Razorpay integration exists in the codebase
- Checkout uses a simple HTML form POST → PHP processes order → redirect
- Orders table has `payment_method` (VARCHAR 50) and `payment_status` (ENUM: pending/paid/failed)
- No `razorpay_payment_id` column exists yet
- `.env` file + `config/env.php` already exist for environment variables
- CSRF protection via `generate_csrf_token()` in `includes/security.php`

## Plan

### Step 1: Add Razorpay Test Keys to `.env`
**File:** `.env`
```
RAZORPAY_KEY_ID=rzp_test_XXXXXXXXXXXX
RAZORPAY_KEY_SECRET=XXXXXXXXXXXXXXXXXXXXXXXX
```
User needs to create a Razorpay test account and paste their keys here.

### Step 2: Add Razorpay SDK Script
**File:** `includes/header.php`
- Add `<script src="https://checkout.razorpay.com/v1/checkout.js"></script>` before `</head>`

### Step 3: Add `razorpay_payment_id` Column to Orders Table
**File:** Run SQL (or add to `database/wolf_nutrition.sql`)
```sql
ALTER TABLE orders ADD COLUMN razorpay_payment_id VARCHAR(100) DEFAULT NULL AFTER payment_status;
```

### Step 4: Modify `checkout.php` - Payment Flow
**File:** `checkout.php`

**Before (current flow):**
1. Form POST → PHP validates → creates order → sets `payment_status = 'paid'` for UPI/CARD → redirect

**After (new flow):**
1. Form is intercepted by JavaScript (no direct POST for UPI/CARD)
2. JS collects form data, opens Razorpay checkout modal
3. On success callback: sends `razorpay_payment_id` + form data to PHP via fetch POST
4. PHP creates order with `razorpay_payment_id` and `payment_status = 'paid'`
5. COD still works as a direct form POST (no Razorpay modal)

**Key changes in checkout.php:**

a) Add Razorpay key ID as a JS variable:
```php
$razorpay_key = getenv('RAZORPAY_KEY_ID');
```

b) Modify the "PLACE ORDER NOW" button behavior:
- For UPI/CARD: `onclick="initRazorpay(event)"` — opens Razorpay modal
- For COD: normal form submit (unchanged)

c) Add new JS function `initRazorpay(e)`:
- Prevents default form submit
- Validates required fields client-side
- Calls `fetch('create_razorpay_order.php', ...)` to get a Razorpay order ID from server
- Opens `Razorpay({ ... })` modal with the order ID
- On success: sends payment_id + all form data to `verify_razorpay.php`
- On failure: shows error alert

d) Add a hidden input for `razorpay_payment_id` that gets populated on success

e) Modify PHP form handler:
- For UPI/CARD: expect `razorpay_payment_id` in POST
- Set `payment_status = 'paid'` and save `razorpay_payment_id`
- For COD: unchanged (payment_status = 'pending')

### Step 5: Create `create_razorpay_order.php`
**New File:** `create_razorpay_order.php`

This endpoint creates a Razorpay order server-side:
- Accepts POST with `amount` (in INR)
- Uses Razorpay API (curl) to create an order: `POST https://api.razorpay.com/v1/orders`
- Auth: Basic auth with `key_id:key_secret`
- Returns JSON: `{ razorpay_order_id: "...", amount: ..., currency: "INR" }`
- CSRF protected

### Step 6: Create `verify_razorpay.php`
**New File:** `verify_razorpay.php`

This endpoint verifies payment after Razorpay success:
- Accepts POST: `razorpay_order_id`, `razorpay_payment_id`, `razorpay_signature`, `csrf_token`
- Verifies signature using HMAC SHA256: `HMAC_SHA256(order_id + "|" + payment_id, key_secret)`
- If signature matches: returns `{ success: true }`
- If not: returns `{ success: false, message: "Payment verification failed" }`

### Step 7: Update Order Insert Query in `checkout.php`
- Add `razorpay_payment_id` column to INSERT statement
- Pass the payment ID from POST data
